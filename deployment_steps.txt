=========================================================
      CHATSPHERE DEPLOYMENT GUIDE (MERN + Socket.io)
=========================================================

This guide will walk you through deploying your ChatSphere application using Vercel (for frontend) and Render (for backend), which are free and easy to use.

---------------------------------------------------------
STEP 1: PREPARE YOUR CODE
---------------------------------------------------------
1. Push your entire project to a single GitHub repository.
2. Ensure your backend dynamically handles the frontend's deployed URL.
   - In `server/index.js`, update your CORS policy so it allows requests from your future Vercel URL.
   - Example: cors: { origin: ["http://localhost:5174", "https://yourfrontend.vercel.app"] }
3. Update your frontend to connect to the backend's deployed URL dynamically.
   - In `client/src/socket.js`, change `'http://localhost:5000'` to your future backend URL (e.g. `'https://yourbackend.onrender.com'`).
   - In `client/src/pages/Rooms.jsx` and `Chat.jsx`, change `http://localhost:5000` fetches to the same backend URL.
   - TIP: Create a file `client/.env` that stores `VITE_API_URL=https://yourbackend.onrender.com` and use `import.meta.env.VITE_API_URL` instead of hardcoded localhost.

---------------------------------------------------------
STEP 2: SET UP MONGODB ATLAS (Cloud Database)
---------------------------------------------------------
1. Go to https://www.mongodb.com/atlas/database and create a free cluster.
2. In "Database Access", create a new database user and password.
3. In "Network Access", add IP address `0.0.0.0/0` (Allow access from anywhere).
4. Click "Connect" -> "Drivers" to get your connection string.
   - It will look like: `mongodb+srv://<username>:<password>@cluster.xyz.mongodb.net/chatdb?retryWrites=true&w=majority`

---------------------------------------------------------
STEP 3: DEPLOY BACKEND ON RENDER
---------------------------------------------------------
1. Go to https://render.com/ and sign up/in with GitHub.
2. Click "New +" and select "Web Service" -> "Build and deploy from a Git repository".
3. Connect your GitHub account and select your repository.
4. Settings:
   - Root Directory: `server`
   - Runtime: `Node`
   - Build Command: `npm install`
   - Start Command: `node index.js`
5. Environment Variables (Environment section):
   - Add `MONGO_URI` and paste your MongoDB Atlas connection string from Step 2.
6. Click "Create Web Service". Wait for it to deploy successfully and copy its public URL (e.g., `https://your-backend.onrender.com`).

---------------------------------------------------------
STEP 4: UPDATE FRONTEND WITH BACKEND URL
---------------------------------------------------------
1. Now that your backend is deployed, go back to your local code.
2. Replace `http://localhost:5000` with the actual Render URL everywhere in your `client` folder (API fetch requests and socket connection).
3. Commit and push these changes to GitHub.

---------------------------------------------------------
STEP 5: DEPLOY FRONTEND ON VERCEL
---------------------------------------------------------
1. Go to https://vercel.com/ and sign up/in with GitHub.
2. Click "Add New..." -> "Project".
3. Import your GitHub repository.
4. Settings:
   - Framework Preset: `Vite`
   - Root Directory: `client`
5. Click "Deploy".
6. Once deployed, Vercel will give you a public URL (e.g., `https://chatsphere.vercel.app`).

---------------------------------------------------------
STEP 6: FINAL CORS CONFIGURATION
---------------------------------------------------------
1. Go to your backend code one last time.
2. Ensure the `cors` setting in `server/index.js` includes your new Vercel frontend URL.
3. Commit and push to GitHub. Render will auto-deploy the change.

You are done! 🎉 
Your real-time chat app is now live and accessible on the internet.
